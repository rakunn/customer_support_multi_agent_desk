# Showcase Copy

## LinkedIn Post

I built a multi-agent Customer Support Agent Desk that automates real support workflows.

Architecture:
- Triage Agent routes requests
- FAQ Agent answers policy questions with local RAG
- Order Agent checks mock order data
- Refund Agent evaluates refund eligibility
- Escalation Agent prepares human handoffs
- Quality Review Agent checks final responses

Key features:
- Specialist handoffs
- Tool use against customer/order/ticket data
- Human approval for risky refunds
- Guardrails for prompt injection and policy violations
- Evaluation suite with labeled support tickets
- Trace timeline for debugging

This is the kind of AI automation I can build for e-commerce, SaaS, and support teams.

## Upwork Case Study Summary

Built a production-style AI customer support desk that routes incoming messages to specialist agents for FAQs, order tracking, refunds, and technical issues. The system uses tool calling, support-policy retrieval, human approvals for high-risk refunds, guardrails, and evaluation tests. Designed for e-commerce and SaaS teams that want to automate first-line support without losing control over sensitive actions.
